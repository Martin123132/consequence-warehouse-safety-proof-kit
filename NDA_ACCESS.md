# NDA Access Path

The private engine, full evidence runs, implementation details, benchmark
internals, and unreleased product plans may be protected as private confidential
materials.

## NDA May Be Required For

- private repository access
- source code review
- private benchmark internals
- full generated evidence bundles
- customer-specific pilot design
- commercial evaluation
- hosted-service or AI-platform integration review
- roadmap, pricing, or deployment discussion

## Suggested Flow

1. Review the public proof kit.
2. Submit the access request checklist.
3. Confirm intended use and access level.
4. Sign NDA or evaluation terms if required.
5. Receive limited private access or a guided review.
6. Revoke access when the review ends unless a written agreement extends it.

GitHub collaborator access is only an access mechanism. It is not a licence,
commercial permission, or waiver of confidentiality.
