# Evidence Review Guide

Start here if you are reviewing the proof kit for diligence, technical interest,
or private access discussion.

## Recommended Order

1. Read the [README](../README.md).
2. Read the [public proof one-pager](public-proof-one-pager.md).
3. Inspect the [warehouse evidence card](../proofs/v1.6-warehouse-evidence/warehouse/evidence_card.md).
4. Inspect the [machine-readable metrics summary](../proofs/v1.6-warehouse-evidence/metrics_summary.json).
5. Read the [verification notes](../proofs/v1.6-warehouse-evidence/verification_notes.md).
6. Review the [proof scope](../PROOF_SCOPE.md).
7. Use the [access request checklist](access-request-checklist.md) if you need private review.

## Evidence Classes

This public kit contains:

- summarized metrics
- source hash references
- proof cards
- architecture overview
- interface screenshots
- access and licence boundary documents

It does not contain:

- private source code
- notebook source
- raw generated evidence runs
- implementation internals
- customer data

## Review Questions

Useful questions for a reviewer:

- Does the locked scenario describe a plausible disruption?
- Do the headline metrics address delivery, harm, and energy together?
- Are the limits of the proof stated clearly?
- Is the public/private boundary explicit enough?
- Is private access needed for a deeper technical review?
